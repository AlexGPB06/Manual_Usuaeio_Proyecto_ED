Descripción de Clases Java

Componentes clave que dan vida al sistema del Banco Financiero.

Descripción: Es el punto de entrada y controlador para la versión con Interfaz Gráfica de Usuario (GUI) del proyecto.

Uso en el Programa: Reemplaza la consola por una experiencia visual. Dibuja las ventanas, botones y campos de texto, y captura las acciones del usuario (clics) para ejecutar la lógica del banco.

Métodos Clave y su Aplicación:
  • main(): Inicia la aplicación gráfica, creando y mostrando la ventana principal del programa.
  • crearLoginPanel() / crearMainMenuPanel(): Se encargan de construir las diferentes pantallas de la interfaz (la de inicio de sesión y el menú principal del cliente).
  • intentarLogin(): Se ejecuta cuando el usuario hace clic en el botón "Iniciar Sesión". Toma los datos de los campos de texto y los valida.
  • accionDepositar() / accionRetirar(): Son los manejadores de eventos para los botones de operaciones. Por ejemplo, accionDepositar() se activa al pulsar "Realizar Depósito" y muestra el diálogo para introducir el monto.

Descripción: Es la clase que actúa como modelo de datos. Representa la estructura y la información de un cliente.

Uso en el Programa: Es el "plano" para cada usuario. Cada objeto Cliente almacena de forma organizada todos los datos de un usuario, que son manipulados por el resto del sistema.

Atributos y Métodos Clave:
  • Atributos (ID, Nombre, Monto, etc.): Son utilizados constantemente para mostrar la información del cliente en pantalla (en la consola o en la GUI) y para realizar cálculos en las transacciones.
  • Depositar(monto): Este método es llamado directamente desde los menús (Main y BancoUI) para actualizar el saldo del cliente cuando se realiza un depósito.

Descripción: Estructura de datos principal para almacenar a todos los clientes, usando el número de tarjeta como clave.

Uso en el Programa: Es la "base de datos" del sistema, permitiendo un acceso y validación de usuarios extremadamente rápido.

Métodos Clave y su Aplicación:
  • put(clave, valor): Se utiliza al registrar un nuevo cliente para guardarlo en el sistema, asociando su número de tarjeta (clave) con su objeto Cliente (valor).
  • get(clave): Es fundamental en el inicio de sesión y en las transferencias. Permite buscar y obtener los datos de un cliente de forma instantánea solo conociendo su número de tarjeta.
  • contains(clave): Se usa durante el proceso de registro para verificar si un número de tarjeta ya existe y así evitar clientes duplicados.

Descripción: Implementa una estructura de datos LIFO (Last-In, First-Out).

Uso en el Programa: Es el motor del Historial de Movimientos, mostrando siempre la transacción más reciente primero.

Métodos Clave y su Aplicación:
  • push(dato): Se llama después de cada operación exitosa (depósito, retiro, transferencia) para "apilar" un texto descriptivo de la transacción en el historial del cliente.
  • showAll(): Se utiliza cuando el usuario selecciona la opción de ver su historial, para mostrar en pantalla todos los movimientos almacenados, del más reciente al más antiguo.

Descripción: Implementa una estructura de datos FIFO (First-In, First-Out), donde el primer elemento en entrar es el primero en salir.

Uso en el Programa: Gestiona las transferencias para que puedan ser procesadas en el orden en que se solicitaron.

Métodos Clave y su Aplicación:
  • enqueue(monto): Se llama cada vez que se realiza una transferencia exitosa, añadiendo el monto transferido al final de la cola.
  • dequeue(): Se utiliza en la opción "Realizar Transferencias" del menú de consola para procesar y eliminar la transacción más antigua de la cola.

Descripción: Node es la clase que representa un eslabón individual, conteniendo un dato y enlaces al nodo siguiente y anterior. LinkedList es la clase que gestiona estos nodos para formar una lista enlazada.

Uso en el Programa: Son las estructuras fundamentales de bajo nivel sobre las que se construyen Stack y Queue. En lugar de usar las listas predefinidas de Java, el proyecto utiliza estas clases personalizadas para demostrar el funcionamiento interno de las estructuras de datos.

Métodos Clave y su Aplicación:
  • Node(valData): Es el constructor que se usa en Stack y Queue para crear un nuevo "contenedor" cada vez que se añade un elemento.
  • printList(): Método de LinkedList que permite recorrer e imprimir el contenido de la lista, usado por Queue para mostrar las transferencias pendientes.

Descripción: Implementa un Árbol Binario de Búsqueda para organizar valores numéricos de forma ordenada.

Uso en el Programa: Se utiliza como una herramienta de ordenamiento bajo demanda para el historial de transacciones.

Métodos Clave y su Aplicación:
  • insertarCliente(clienteFalso, "Historial"): Aunque el nombre dice "Cliente", se usa de forma ingeniosa. Cuando el usuario quiere ordenar su historial, este método se llama repetidamente para insertar cada monto de transacción en el árbol.
  • mostrarIdsEnOrden(): Inmediatamente después de que el árbol se ha llenado con los montos, se llama a este método para recorrer el árbol e imprimir los montos en orden ascendente en la pantalla.

Descripción: Es una clase de utilidad que contiene métodos estáticos para verificar la validez y el tipo de un número de tarjeta de crédito.

Uso en el Programa: Actúa como una herramienta de seguridad y validación de datos de entrada.

Métodos Clave y su Aplicación:
  • validarTarjeta(numero): Se llama durante el registro y el inicio de sesión. Utiliza el algoritmo de Luhn para confirmar que el número de tarjeta introducido por el usuario es matemáticamente válido.
  • obtenerTipoTarjeta(numero): Se usa después de una validación exitosa para identificar y mostrar la marca de la tarjeta (ej. Visa, MasterCard), mejorando la experiencia del usuario.

Descripción: Clase de utilidad que agrupa algoritmos de recursividad y "Divide y Vencerás".

Aplicación: Se ha integrado en el menú principal para ofrecer una función de análisis: "Ver cliente con mayor saldo".

Métodos Clave y su Aplicación:
  • encontrarClienteMayorSaldo(clientes, inicio, fin): Este es el método de "divide y vencerás" que se llama cuando el usuario selecciona la opción 7 del menú. Divide la lista de clientes a la mitad repetidamente para encontrar de forma eficiente al cliente con el saldo más alto.
  • calcularInteresCompuesto(capital, tasa, tiempo): Es el método recursivo utilizado en la "Caja de Ahorros" para calcular y mostrar la proyección de crecimiento de la inversión del usuario.

Descripción: Clase de utilidad que agrupa diferentes algoritmos de búsqueda para encontrar clientes.

Uso en el Programa: Proporciona la lógica para buscar clientes en el sistema utilizando diferentes criterios y algoritmos, además de comparar su rendimiento.

Métodos Clave y su Aplicación:
  • busquedaLinealPorID() / busquedaLinealPorNombre(): Implementan una búsqueda secuencial simple para encontrar un cliente.
  • busquedaBinariaPorID(): Realiza una búsqueda mucho más eficiente en un arreglo de clientes previamente ordenado, demostrando la ventaja de los algoritmos logarítmicos.
  • compararMetodosBusqueda(): Ejecuta y mide el tiempo de las búsquedas lineal y binaria para mostrar cuál es más rápida en la práctica.

Descripción: Contiene implementaciones de algoritmos de ordenamiento para organizar arreglos de clientes.

Uso en el Programa: Se utiliza para ordenar la lista de clientes según diferentes atributos como el saldo, el ID o el nombre, y para comparar la eficiencia entre algoritmos.

Métodos Clave y su Aplicación:
  • quickSortPorSaldo() / quickSortPorID(): Utiliza el algoritmo QuickSort, conocido por su alta eficiencia en promedio, para ordenar clientes por su saldo o ID.
  • mergeSortPorNombre(): Implementa el algoritmo MergeSort, que es muy estable y eficiente, para ordenar a los clientes alfabéticamente por su nombre.
  • compararAlgoritmos(): Mide y compara los tiempos de ejecución de QuickSort y MergeSort para demostrar sus diferencias de rendimiento.

Descripción: Estas clases implementan estructuras de datos de tipo Grafo para modelar las relaciones entre clientes y monitorear la actividad financiera en busca de anomalías.

Uso en el Programa: Son herramientas avanzadas para el análisis de seguridad y la detección de fraudes.

Métodos Clave y su Aplicación:
  • GrafoBancario.buscarCaminoBFS() / DFS(): Utiliza algoritmos de recorrido de grafos para determinar si existe una ruta de transacciones entre dos clientes.
  • GrafoBancario.detectarCiclos(): Una función crucial para la seguridad, ya que puede identificar patrones de transferencias circulares que a menudo indican lavado de dinero o fraude.
  • GrafoTransacciones.verificarActividadSospechosa(): Es el núcleo del sistema de alerta. Revisa si un cliente realiza demasiadas transacciones en un corto período de tiempo, un indicador común de una tarjeta comprometida.

Descripción: Clases responsables de la persistencia de datos. Se encargan de encontrar, leer y escribir la información de los clientes en el archivo clientes.csv.

Uso en el Programa: Son el puente entre la aplicación y el almacenamiento en disco, asegurando que los datos de los clientes se carguen al iniciar y se guarden al registrar nuevos usuarios.

Métodos Clave y su Aplicación:
  • CSVPathResolver.obtenerRutaCSV(): Aumenta la robustez del programa al buscar el archivo clientes.csv en múltiples directorios, evitando errores si el programa se ejecuta desde diferentes ubicaciones.
  • CSVClientLoader.cargarClientes(): Se ejecuta al inicio para leer el archivo CSV, crear los objetos Cliente y cargarlos en la HashTable del sistema.
  • CSVClientLoader.guardarCliente(): Se llama durante el registro para añadir la información del nuevo cliente al final del archivo CSV.

Descripción: Es una clase de modelo de datos (POJO) que representa una única transacción financiera.

Uso en el Programa: Encapsula toda la información de una operación (ID, tarjeta, monto, tipo, fecha). Estos objetos son utilizados por el GrafoTransacciones para llevar un registro detallado de la actividad de cada cliente.

Métodos Clave y su Aplicación:
  • Transaccion(id, tarjeta, monto, tipo): Es el constructor para crear un nuevo objeto de transacción cada vez que se realiza una operación.
  • Getters (getMonto, getTipo, etc.): Permiten a otras clases, como el sistema de detección de fraude, acceder a los detalles de la transacción para su análisis.
