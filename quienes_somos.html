Descripción de Clases Java

Componentes clave que dan vida al sistema del Banco Financiero.

Descripción: Es el punto de entrada y el cerebro de la versión de consola de la aplicación. Orquesta todo el flujo del programa.

Uso en el Programa: Inicia el sistema, carga los clientes iniciales y presenta los menús de texto al usuario para que interactúe.

Métodos Clave y su Aplicación:
  • main(): Es el primer método que se ejecuta. Su función es arrancar la aplicación, crear la tabla hash de clientes y mostrar el menú de bienvenida (login/registro).
  • mostrarMenuDelBanco(): Se activa después de un inicio de sesión exitoso. Contiene el bucle principal con las opciones bancarias (depositar, retirar, etc.) y gestiona las llamadas a las funciones correspondientes.
  • iniciarSesion() / registrarCliente(): Se utilizan en el menú de bienvenida para controlar el acceso y la creación de nuevos usuarios, validando los datos introducidos por el usuario.

Descripción: Es el punto de entrada y controlador para la versión con Interfaz Gráfica de Usuario (GUI) del proyecto.

Uso en el Programa: Reemplaza la consola por una experiencia visual. Dibuja las ventanas, botones y campos de texto, y captura las acciones del usuario (clics) para ejecutar la lógica del banco.

Métodos Clave y su Aplicación:
  • main(): Inicia la aplicación gráfica, creando y mostrando la ventana principal del programa.
  • crearLoginPanel() / crearMainMenuPanel(): Se encargan de construir las diferentes pantallas de la interfaz (la de inicio de sesión y el menú principal del cliente).
  • intentarLogin(): Se ejecuta cuando el usuario hace clic en el botón "Iniciar Sesión". Toma los datos de los campos de texto y los valida.
  • accionDepositar() / accionRetirar(): Son los manejadores de eventos para los botones de operaciones. Por ejemplo, accionDepositar() se activa al pulsar "Realizar Depósito" y muestra el diálogo para introducir el monto.

Descripción: Es la clase que actúa como modelo de datos. Representa la estructura y la información de un cliente.

Uso en el Programa: Es el "plano" para cada usuario. Cada objeto Cliente almacena de forma organizada todos los datos de un usuario, que son manipulados por el resto del sistema.

Atributos y Métodos Clave:
  • Atributos (ID, Nombre, Monto, etc.): Son utilizados constantemente para mostrar la información del cliente en pantalla (en la consola o en la GUI) y para realizar cálculos en las transacciones.
  • Depositar(monto): Este método es llamado directamente desde los menús (Main y BancoUI) para actualizar el saldo del cliente cuando se realiza un depósito.

Descripción: Estructura de datos principal para almacenar a todos los clientes, usando el número de tarjeta como clave.

Uso en el Programa: Es la "base de datos" del sistema, permitiendo un acceso y validación de usuarios extremadamente rápido.

Métodos Clave y su Aplicación:
  • put(clave, valor): Se utiliza al registrar un nuevo cliente para guardarlo en el sistema, asociando su número de tarjeta (clave) con su objeto Cliente (valor).
  • get(clave): Es fundamental en el inicio de sesión y en las transferencias. Permite buscar y obtener los datos de un cliente de forma instantánea solo conociendo su número de tarjeta.
  • contains(clave): Se usa durante el proceso de registro para verificar si un número de tarjeta ya existe y así evitar clientes duplicados.

Descripción: Implementa una estructura de datos LIFO (Last-In, First-Out).

Uso en el Programa: Es el motor del Historial de Movimientos, mostrando siempre la transacción más reciente primero.

Métodos Clave y su Aplicación:
  • push(dato): Se llama después de cada operación exitosa (depósito, retiro, transferencia) para "apilar" un texto descriptivo de la transacción en el historial del cliente.
  • showAll(): Se utiliza cuando el usuario selecciona la opción de ver su historial, para mostrar en pantalla todos los movimientos almacenados, del más reciente al más antiguo.

Descripción: Implementa una estructura de datos FIFO (First-In, First-Out), donde el primer elemento en entrar es el primero en salir.

Uso en el Programa: Gestiona las transferencias para que puedan ser procesadas en el orden en que se solicitaron.

Métodos Clave y su Aplicación:
  • enqueue(monto): Se llama cada vez que se realiza una transferencia exitosa, añadiendo el monto transferido al final de la cola.
  • dequeue(): Se utiliza en la opción "Realizar Transferencias" del menú de consola para procesar y eliminar la transacción más antigua de la cola.
