Descripción de Clases Java

Componentes clave que dan vida al sistema del Banco Financiero.

Descripción: Es el cerebro y el punto de entrada de toda la aplicación. Se encarga de controlar los menús, gestionar el flujo del programa y coordinar las interacciones entre el usuario y las estructuras de datos.

Contribución: Actúa como el "director de orquesta", inicializando los datos de los clientes y llamando a las funciones correspondientes según las selecciones del usuario.

Métodos Principales:
  • main(): Inicia el programa y muestra el menú de bienvenida (login/registro).
  • mostrarMenuDelBanco(): Gestiona el menú principal del usuario una vez que ha iniciado sesión, conteniendo toda la lógica de las operaciones bancarias.
  • iniciarSesion() / registrarCliente(): Controlan los flujos de autenticación y creación de nuevos usuarios.

Descripción: Es una clase de tipo "modelo" (POJO) que representa la estructura de un cliente del banco. Almacena toda la información personal y financiera de un usuario.

Contribución: Sirve como el plano o plantilla para cada usuario del sistema. Sin esta clase, no habría forma de almacenar de manera organizada los datos de los clientes.

Atributos Principales:
  • ID: Identificador único para cada cliente.
  • Nombre: Nombre completo del cliente.
  • Monto: Saldo en la cuenta principal.
  • montoAhorros: Saldo guardado en la Caja de Ahorros.

Descripción: Es la estructura de datos principal para almacenar a todos los clientes. Utiliza el número de tarjeta como "clave" para guardar y acceder a cada objeto `Cliente`.

Contribución: Proporciona un acceso casi instantáneo a la información de cualquier cliente, lo que hace que el inicio de sesión y la validación de transferencias sean extremadamente rápidos y eficientes.

Métodos Principales:
  • put(clave, valor): Inserta un nuevo cliente en la tabla usando su tarjeta como clave.
  • get(clave): Busca y devuelve un cliente a partir de su número de tarjeta.

Descripción: Implementa una estructura de datos LIFO (Last-In, First-Out), es decir, el último elemento que entra es el primero que sale.

Contribución: Se utiliza para gestionar el **Historial de Movimientos**. Cada nueva transacción se "apila" encima de la anterior, por lo que al mostrar el historial, se ven las operaciones más recientes primero, que es el comportamiento esperado.

Métodos Principales:
  • push(dato): Añade una nueva transacción al tope de la pila.
  • showAll(): Muestra todos los elementos desde el más reciente al más antiguo.

Descripción: Implementa un Árbol Binario de Búsqueda diseñado para organizar objetos `Cliente`.

Contribución: Su principal función en el menú principal es la de **herramienta de ordenamiento bajo demanda**. En el historial, se crea un árbol temporal para ordenar los montos de las transacciones de forma muy eficiente sin tener que usar un algoritmo de ordenamiento sobre un arreglo.

Métodos Principales:
  • insertarCliente(cliente): Inserta un cliente en el árbol, manteniéndolo ordenado por ID.
  • mostrarIdsEnOrden(): Recorre el árbol e imprime los IDs (usados para los montos) en orden ascendente.
